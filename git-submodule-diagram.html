¿Qué es un Git Submodule?

Un Git Submodule es una forma de incluir un repositorio Git dentro de otro repositorio Git. En este proyecto, el submodule bd-relacional contiene toda la documentación y código del sistema de inventario, manteniéndose como un proyecto independiente pero integrado.

Comandos Principales
# Agregar submodule
git submodule add [URL] [path]
# Actualizar submodule
git submodule update --remote
# Clonar con submodules
git clone --recursive [URL]
Beneficios Clave
Control de versiones independiente
El submodule mantiene su propio historial
Código modular y reutilizable
Puede usarse en otros proyectos
Fácil mantenimiento
Actualizaciones independientes
Colaboración eficiente
Equipos pueden trabajar en módulos específicos
